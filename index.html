A). API Section

Before start using the application need to complete API setup, follow below steps to activate API from the WordPress backend

  1. Copy API.zip file that you get with application
  2. Put API folder at WordPress backend at 'wp-content\plugins\school-management\' and extract it
  3. Add this line "require_once SMS_PLUGIN_DIR . '/api/school-api-files.php';" at "wp-content\plugins\school-management\school-management.php""
  4. After following the above steps API will be activated

B). Setting up android project


1. Requirement
  1. Latest Android Studio
  2. Windows XP (32-bit), Vista (32- or 64-bit), or Windows 7 (32- or 64-bit)
  3. Mac OS X 10.5.8 or later (x86 only)
  4. Linux (tested on Ubuntu Linux, Lucid Lynx)
  5. GNU C Library (glibc) 2.7 or later is required.
  6. On Ubuntu Linux, version 8.04 or later is required.64-bit distributions must be capable of running 32-bit applications.
  7. 4 GB RAM minimum, 8 GB RAM recommended.
  8. At least 1 GB for Android SDK, emulator system images, and caches.
  9. Oracle® Java Development Kit (JDK) 8.
2. Installing the latest version of Android Studio.
School MasterApp is built with the latest Android studio Arctic Fox (2020.3.1)
NOTE: There are several other tools to build and run android projects.
  1. Go to the Android Developer Page
  2. Download the Android Studio. This should contain the latest version of Android Studio
  3. Install Android studio. more information Go to install Android studio
3. Update the IDE and SDK Tools
  1. The Android SDK Manager provides the SDK tools, platforms, and other components you need to develop your apps.
  2. School MasterApp minimum required SDK Platform API level 21
  3. Install SDK Platform API levels.
  4. Open the Preferences window by clicking File -> Settings.
  5. In the left panel, click Appearance & Behavior -> System Settings -> Android SDK.
  6. Select required SDK platforms
  7. Click ok.
4. Project Configuration
  1. On Android Studio menu select File -> New -> Import Project
  2. In the Select Eclipse or Gradle Project to Import window that appears, navigate to the root directory of the project you want to import.
  3. Click OK.
  4. Android Studio then open the project in a new IDE window.

C).Set Up Firebase

To set up Firebase Database for a School App, follow these step-by-step instructions:

Step 1: Create a Firebase project

  • Go to the Firebase console (https://console.firebase.google.com/) and sign in with your Google account.
  • Click on "Add project" or select an existing project if you have one.
  • Enter a name for your project and optionally choose a country/region.
  • Click on "Create Project" to create the Firebase project.

    • Step 2: Connect your app to Firebase

  • In the Firebase console, go to your project.
  • Click on the "Add app" button to add your Android app.
  • Follow the instructions to register your app, providing the app package name (usually "com.example.your_app") and app nickname.
  • Download the google-services.json file and place it in the android/app directory of your Android project.

    • D).Firebase Cloud Messaging Set Up Key

  • Go to Firebase Console, select the project setting then go to Cloud Messaging.
  • Get the server key and login in web with admin credentials.
  • Select the General Settings in System Settings option. Paste the FCM Key in Push Notification Setting column.
  • FCM Key is set now, push notification will work after this set up.

    The application name can be changed from the strings.xml file in the values folder.

    F). Changing Package name

    Change the package name in AndroidManifest.xml and Gradle Scripts file.
    Every application in Google Play has an unique application name usually in the form of com.example.app. Before you can publish your application you need to change the application name to something unique (like use your domain as prefix in reverse order). To change it, you need to:

    1. Run IDE and open the project
    2. click on manifest folder and open AndroidManifest.xml file
    3. Change the package value.
    4. click on the Gradle Scripts
    5. open build.gradle (Module:app) file -> android -> defaultConfig -> applicationId value Rename
    6. click on Sync Now

    I).Sign your release build


    You can use Android Studio to manually generate signed APKs, either one at a time, or for multiple build variants at once. Instead of manually signing APKs, you can also configure your Gradle build settings to handle signing automatically during the build process. This section describes the manual signing process.more information to see,Configure the Build Process to Automatically Sign Your APK.

    To manually sign your APK for release in Android Studio, follow these steps:
    1. In the menu bar, click Build -> Generate Signed APK.
    2. Select the module you would like to release from the drop down, and click Next
    3. If you already have a keystore, go to step 12. If you want to create a new keystore, click Create new
    4. On the New Key Store window, provide the following information for your keystore and key
    5. Key store path: Select the location where your keystore should be created.
    6. Password: Create and confirm a secure password for your keystore.
    7. Alias: Enter an identifying name for your key.
    8. Password: Create and confirm a secure password for your key. This should be different from the password you chose for your keystore
    9. Validity (years): Set the length of time in years that your key will be valid. Your key should be valid for at least 25 years, so you can sign app updates with the same key through the lifespan of your app.
    10. Certificate: Enter some information about yourself for your certificate. This information is not displayed in your app, but is included in your certificate as part of the APK.
    11. Once you complete the form, click OK.
    12. On the Generate Signed APK Wizard window, select a keystore, a private key, and enter the passwords for both. (If you created your keystore in the last step, these fields are already populated for you.) Then click Next.
    13. On the next window,select a destination for the signed APK(s), select the build type, (if applicable) choose the product flavor(s), and click Finish.

    J).Release App in PlayStore

    To sign a release app for School Management , you can follow these steps:

  • Please be aware that there is a registration fee of at least $25 associated with the process.
  • Click on the Create app button in the All apps section. Enter the app details in the "Create app" form.
  • After creating the app, we have to go through multiple tasks to release the application.
  • However, all these steps will be self-explanatory.
  • At each step, you'll be prompted for information about your application, advertisements, privacy policy, and more.
  • It's important to note that you can proceed to publish only after completing these steps.
  • Make sure to mention everything clearly. Especially the questions about data collection. Because, at any time if Google found that your prescribed information is wrong, your app will not be published or will be taken out, if it's published.
  • After completing these steps, we have to send the app for review. Our app will be published once the review is completed from Google Play team.
  • It's important to note that there's a higher chance of the app getting rejected if it has security or advertising-related issues.
  • Therefore, make sure to thoroughly review your app to ensure that it doesn't have any potential problems in data collection. This will contribute to a smoother and more successful app release experience.
